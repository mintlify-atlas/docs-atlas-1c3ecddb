## Endpoint

```
GET /api/admin/dlq
```

Retrieve a paginated list of failed extraction jobs from the Dead Letter Queue (DLQ). Jobs enter the DLQ after exhausting all retry attempts (3 attempts with exponential backoff).

## Authentication

Requires a valid Bearer token in the `Authorization` header.

```bash
Authorization: Bearer YOUR_API_KEY
```

<Warning>
  This is an admin endpoint. Ensure proper access controls are in place in production.
</Warning>

## Query Parameters

<ParamField query="start" type="number" default="0">
  Starting index for pagination (0-based).
</ParamField>

<ParamField query="end" type="number" default="20">
  Ending index for pagination (inclusive). Maximum 100.
</ParamField>

## Response

<ResponseField name="count" type="number">
  Number of failed jobs returned in this response.
</ResponseField>

<ResponseField name="jobs" type="array">
  Array of failed job objects.
  
  <Expandable title="job properties">
    <ResponseField name="jobId" type="string">
      Unique identifier of the failed job.
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Job type: `"single_message"` or `"chat_log"`.
    </ResponseField>
    
    <ResponseField name="orgId" type="string">
      Organization ID that owns this job.
    </ResponseField>
    
    <ResponseField name="error" type="string">
      Failure reason from the last attempt.
    </ResponseField>
    
    <ResponseField name="attempts" type="number">
      Number of processing attempts made (typically 3).
    </ResponseField>
    
    <ResponseField name="failedAt" type="string">
      ISO 8601 timestamp when the job finally failed.
    </ResponseField>
    
    <ResponseField name="correlationId" type="string">
      Correlation ID for tracing this job through logs.
    </ResponseField>
  </Expandable>
</ResponseField>

## Common Failure Reasons

<AccordionGroup>
  <Accordion title="Anthropic API Errors">
    - `Anthropic API rate limit exceeded` - Too many requests
    - `Anthropic API error: 503 Service Unavailable` - Temporary API outage
    - `Invalid API key` - Authentication failure
    
    **Action**: Wait for rate limit reset or check API key validity
  </Accordion>
  
  <Accordion title="Database Errors">
    - `Database connection timeout` - DB connection pool exhausted
    - `Unique constraint violation` - Duplicate order ID
    - `Query timeout` - Slow database performance
    
    **Action**: Check database health, connection pool settings
  </Accordion>
  
  <Accordion title="Validation Errors">
    - `Invalid message format` - Malformed input data
    - `Missing required field` - Schema validation failure
    - `Invalid orgId` - Organization not found
    
    **Action**: These jobs cannot be retried without fixing the input data
  </Accordion>
  
  <Accordion title="Timeout Errors">
    - `Job processing timeout` - Exceeded maximum processing time
    - `Worker shutdown during processing` - Server restart
    
    **Action**: Safe to retry
  </Accordion>
</AccordionGroup>

## Pagination Example

```javascript
const fetchAllFailedJobs = async () => {
  const allJobs = [];
  let start = 0;
  const pageSize = 20;
  
  while (true) {
    const response = await fetch(
      `https://your-domain.com/api/admin/dlq?start=${start}&end=${start + pageSize}`,
      {
        headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` }
      }
    );
    
    const data = await response.json();
    allJobs.push(...data.jobs);
    
    // No more jobs
    if (data.count < pageSize) break;
    
    start += pageSize;
  }
  
  return allJobs;
};
```

## Monitoring & Alerting

Set up alerts when the DLQ grows:

```javascript
const checkDLQSize = async () => {
  const response = await fetch(
    'https://your-domain.com/api/admin/dlq?start=0&end=1',
    {
      headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` }
    }
  );
  
  const data = await response.json();
  
  // Check queue health for total failed count
  const healthResponse = await fetch(
    'https://your-domain.com/api/queue/health',
    {
      headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` }
    }
  );
  
  const health = await healthResponse.json();
  const failedCount = health.extraction.failed;
  
  if (failedCount > 10) {
    console.error(`🚨 ${failedCount} jobs in DLQ! Manual intervention needed.`);
    // Send alert to on-call engineer
  }
  
  return failedCount;
};
```

## Rate Limiting

This endpoint is subject to rate limiting:
- **Limit**: 20 requests per minute per API key
- **Quota**: Enforced via the `generalLimiter` middleware

## Notes

<Info>
  Failed jobs are retained indefinitely in the DLQ until manually retried or removed. Monitor this queue regularly to prevent unbounded growth.
</Info>

<Tip>
  Use the `correlationId` to trace failed jobs through your logging system (Sentry, DataDog, etc.) for detailed debugging.
</Tip>

<Warning>
  Some failures are permanent (e.g., validation errors, invalid API keys). Analyze the error message before retrying to avoid wasting resources.
</Warning>
