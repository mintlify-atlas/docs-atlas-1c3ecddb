title: "Generate Invoice"

---

## Endpoint

```
POST /api/generate-invoice
```

Generates a GST-compliant invoice for a confirmed order, creates a PDF file, and uploads it to Azure Blob Storage.

## Authentication

Requires a valid Bearer token in the `Authorization` header.

```bash
Authorization: Bearer YOUR_API_KEY
```

## Request Body

<ParamField body="orderId" type="string" required>
  The unique identifier of the order to generate an invoice for.
</ParamField>

## Headers

<ParamField header="Idempotency-Key" type="string">
  Optional idempotency key to prevent duplicate invoice generation. If provided, subsequent requests with the same key will return the cached response.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message indicating invoice generation status.
</ResponseField>

<ResponseField name="invoice" type="object">
  The generated invoice object.
  
  <Expandable title="invoice properties">
    <ResponseField name="invoice_number" type="string">
      Sequential invoice number (e.g., "INV-2024-0001").
    </ResponseField>
    
    <ResponseField name="invoice_date" type="string">
      ISO 8601 timestamp of invoice generation.
    </ResponseField>
    
    <ResponseField name="customer_name" type="string">
      Customer name from the order.
    </ResponseField>
    
    <ResponseField name="items" type="array">
      Array of invoice line items.
    </ResponseField>
    
    <ResponseField name="subtotal" type="number">
      Subtotal before tax.
    </ResponseField>
    
    <ResponseField name="tax_amount" type="number">
      GST tax amount.
    </ResponseField>
    
    <ResponseField name="total" type="number">
      Total amount including tax.
    </ResponseField>
    
    <ResponseField name="gst_number" type="string">
      Business GST registration number.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="downloadEndpoint" type="string">
  API endpoint to download the generated PDF invoice.
</ResponseField>

## Example Request

```bash
curl -X POST https://your-domain.com/api/generate-invoice \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: unique-key-123" \
  -d '{
    "orderId": "cm59h2c4y000008l31y4q8bqz"
  }'
```

## Notes

<Note>
  Invoice numbers are automatically assigned in sequential order per organization. The sequence is managed at the database level to prevent duplicates.
</Note>

<Warning>
  Ensure the order exists and belongs to your organization before generating an invoice. The order must be in a valid state.
</Warning>

## Rate Limiting

This endpoint is subject to rate limiting:
- **Limit**: 10 requests per minute per API key
- **Quota**: Enforced via the `extractLimiter` middleware

## Idempotency

This endpoint supports idempotency. Include an `Idempotency-Key` header to safely retry requests without creating duplicate invoices. The same key will return the cached response for 24 hours.
