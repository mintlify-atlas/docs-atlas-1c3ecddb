---
title: "List Orders"
api: "GET /api/orders"
description: "Retrieve a paginated list of orders for your organization"
---

## Authentication

Requires a valid API key in the `Authorization` header.

## Query Parameters

<ParamField query="limit" type="number" default="50">
  Maximum number of orders to return per request. Range: 1-100.
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of orders to skip before starting to return results. Used for pagination.
</ParamField>

## Response

Returns an array of order objects. Each order includes customer details, line items, delivery information, and order status.

<ResponseField name="id" type="string">
  Unique order identifier (UUID)
</ResponseField>

<ResponseField name="customer_name" type="string" nullable>
  Customer name extracted from the chat
</ResponseField>

<ResponseField name="items" type="array">
  Array of order line items
  <Expandable title="Item properties">
    <ResponseField name="product_name" type="string">
      Name of the product
    </ResponseField>
    <ResponseField name="quantity" type="number">
      Quantity ordered (must be positive)
    </ResponseField>
    <ResponseField name="price" type="number" nullable>
      Price per unit (can be null if not specified)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="delivery_address" type="string" nullable>
  Delivery address for the order
</ResponseField>

<ResponseField name="delivery_date" type="string" nullable>
  Requested delivery date (ISO 8601 format)
</ResponseField>

<ResponseField name="special_instructions" type="string" nullable>
  Special instructions or notes from the customer
</ResponseField>

<ResponseField name="total" type="number" nullable>
  Total order amount (numeric)
</ResponseField>

<ResponseField name="confidence" type="string">
  AI extraction confidence level: `high`, `medium`, or `low`
</ResponseField>

<ResponseField name="status" type="string" default="pending">
  Order status: `pending`, `confirmed`, `fulfilled`, or `cancelled`
</ResponseField>

<ResponseField name="created_at" type="string">
  Order creation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="raw_messages" type="array">
  Original chat messages that were used to extract this order
  <Expandable title="Message properties">
    <ResponseField name="sender" type="string">
      Message sender identifier
    </ResponseField>
    <ResponseField name="text" type="string">
      Message content
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="invoice" type="object" nullable>
  Generated invoice details (if invoice has been generated)
  <Expandable title="Invoice properties">
    <ResponseField name="invoice_number" type="string">
      Unique invoice number
    </ResponseField>
    <ResponseField name="date" type="string">
      Invoice date
    </ResponseField>
    <ResponseField name="customer_name" type="string">
      Customer name on invoice
    </ResponseField>
    <ResponseField name="items" type="array">
      Invoice line items with pricing
    </ResponseField>
    <ResponseField name="subtotal" type="number">
      Subtotal before tax
    </ResponseField>
    <ResponseField name="cgst" type="number">
      Central GST amount
    </ResponseField>
    <ResponseField name="sgst" type="number">
      State GST amount
    </ResponseField>
    <ResponseField name="igst" type="number">
      Integrated GST amount (for interstate transactions)
    </ResponseField>
    <ResponseField name="total" type="number">
      Total invoice amount including taxes
    </ResponseField>
    <ResponseField name="business_name" type="string">
      Your business name
    </ResponseField>
    <ResponseField name="gst_number" type="string">
      Your GST registration number
    </ResponseField>
  </Expandable>
</ResponseField>

## Pagination

Use `limit` and `offset` to paginate through results:

- First page: `?limit=50&offset=0`
- Second page: `?limit=50&offset=50`
- Third page: `?limit=50&offset=100`

If fewer orders than `limit` are returned, you've reached the end of the list.

<RequestExample>
```bash cURL
curl -X GET "https://api.chat2cash.ai/api/orders?limit=10&offset=0" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript Node.js
const response = await fetch('https://api.chat2cash.ai/api/orders?limit=10&offset=0', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const orders = await response.json();
console.log(orders);
```

```python Python
import requests

response = requests.get(
    'https://api.chat2cash.ai/api/orders',
    params={'limit': 10, 'offset': 0},
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

orders = response.json()
print(orders)
```
</RequestExample>
