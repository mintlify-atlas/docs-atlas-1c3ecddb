---
title: "Update Order Status"
api: "PATCH /api/orders/:id"
description: "Update the status of an existing order"
---

## Authentication

Requires a valid API key in the `Authorization` header.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the order to update
</ParamField>

## Request Body

<ParamField body="status" type="string" required>
  The new status for the order. Must be one of: `pending`, `confirmed`, `fulfilled`, or `cancelled`
</ParamField>

## Response

Returns the updated order object with the new status.

<ResponseField name="id" type="string">
  Unique order identifier (UUID)
</ResponseField>

<ResponseField name="customer_name" type="string" nullable>
  Customer name extracted from the chat
</ResponseField>

<ResponseField name="items" type="array">
  Array of order line items
  <Expandable title="Item properties">
    <ResponseField name="product_name" type="string">
      Name of the product
    </ResponseField>
    <ResponseField name="quantity" type="number">
      Quantity ordered (must be positive)
    </ResponseField>
    <ResponseField name="price" type="number" nullable>
      Price per unit (can be null if not specified)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="delivery_address" type="string" nullable>
  Delivery address for the order
</ResponseField>

<ResponseField name="delivery_date" type="string" nullable>
  Requested delivery date (ISO 8601 format)
</ResponseField>

<ResponseField name="special_instructions" type="string" nullable>
  Special instructions or notes from the customer
</ResponseField>

<ResponseField name="total" type="number" nullable>
  Total order amount (numeric)
</ResponseField>

<ResponseField name="confidence" type="string">
  AI extraction confidence level: `high`, `medium`, or `low`
</ResponseField>

<ResponseField name="status" type="string">
  Updated order status: `pending`, `confirmed`, `fulfilled`, or `cancelled`
</ResponseField>

<ResponseField name="created_at" type="string">
  Order creation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="raw_messages" type="array">
  Original chat messages that were used to extract this order
  <Expandable title="Message properties">
    <ResponseField name="sender" type="string">
      Message sender identifier
    </ResponseField>
    <ResponseField name="text" type="string">
      Message content
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="invoice" type="object" nullable>
  Generated invoice details (if invoice has been generated)
</ResponseField>

## Status Workflow

Typical order status progression:

1. **pending** - Order extracted from chat, awaiting confirmation
2. **confirmed** - Order confirmed by merchant
3. **fulfilled** - Order completed and delivered
4. **cancelled** - Order cancelled by merchant or customer

<RequestExample>
```bash cURL
curl -X PATCH "https://api.chat2cash.ai/api/orders/ord_7f8a9b2c3d4e5f6g7h8i9j0k" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "confirmed"
  }'
```

```javascript Node.js
const orderId = 'ord_7f8a9b2c3d4e5f6g7h8i9j0k';

const response = await fetch(`https://api.chat2cash.ai/api/orders/${orderId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    status: 'confirmed'
  })
});

const updatedOrder = await response.json();
console.log(updatedOrder);
```

```python Python
import requests

order_id = 'ord_7f8a9b2c3d4e5f6g7h8i9j0k'

response = requests.patch(
    f'https://api.chat2cash.ai/api/orders/{order_id}',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={'status': 'confirmed'}
)

updated_order = response.json()
print(updated_order)
```
</RequestExample>
