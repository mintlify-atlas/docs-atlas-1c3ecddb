## PATCH /api/orders/:id/edit

Update order details including customer name, items, delivery information, and special instructions. Unlike the status-only update endpoint (`/api/orders/:id`), this endpoint allows you to modify the actual order content.

### Authentication

Requires a valid Bearer token (JWT or API key). The order must belong to your organization.

### Rate Limiting

This endpoint uses the extraction rate limiter. Limits vary by organization tier.

### Idempotency

This endpoint supports idempotency via the `Idempotency-Key` header.

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the order to update
</ParamField>

## Request Body

All fields are optional. Only include fields you want to update.

<ParamField body="customer_name" type="string">
  Customer's full name
</ParamField>

<ParamField body="items" type="array">
  Array of order items. This replaces all existing items (delete-then-insert).

  <Expandable title="Item object">
    <ParamField body="product_name" type="string" required>
      Name of the product
    </ParamField>
    <ParamField body="quantity" type="number" required>
      Quantity ordered (must be positive)
    </ParamField>
    <ParamField body="price" type="number">
      Price per unit (must be non-negative)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="delivery_address" type="string">
  Delivery address
</ParamField>

<ParamField body="delivery_date" type="string">
  Delivery date in ISO 8601 format
</ParamField>

<ParamField body="special_instructions" type="string">
  Special instructions or notes for the order
</ParamField>

<ParamField body="total" type="number">
  Total order amount (must be non-negative)
</ParamField>

<Warning>
  The `items` field replaces **all existing items**. If you only want to add or remove specific items, include the complete updated item list in your request.
</Warning>

## Example Request

```bash cURL
curl -X PATCH https://api.chat2cash.com/api/orders/ord_abc123/edit \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "customer_name": "Rajesh Kumar",
    "delivery_address": "123 MG Road, Bangalore 560001",
    "items": [
      {
        "product_name": "Basmati Rice",
        "quantity": 5,
        "price": 80
      },
      {
        "product_name": "Toor Dal",
        "quantity": 2,
        "price": 120
      }
    ],
    "total": 640
  }'
```

```javascript JavaScript
const response = await fetch('https://api.chat2cash.com/api/orders/ord_abc123/edit', {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    customer_name: 'Rajesh Kumar',
    delivery_address: '123 MG Road, Bangalore 560001',
    items: [
      { product_name: 'Basmati Rice', quantity: 5, price: 80 },
      { product_name: 'Toor Dal', quantity: 2, price: 120 }
    ],
    total: 640
  })
});

const order = await response.json();
console.log(order);
```

```python Python
import requests

response = requests.patch(
    'https://api.chat2cash.com/api/orders/ord_abc123/edit',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'customer_name': 'Rajesh Kumar',
        'delivery_address': '123 MG Road, Bangalore 560001',
        'items': [
            {'product_name': 'Basmati Rice', 'quantity': 5, 'price': 80},
            {'product_name': 'Toor Dal', 'quantity': 2, 'price': 120}
        ],
        'total': 640
    }
)

order = response.json()
print(order)
```

## Use Cases

### Update delivery information

```bash
curl -X PATCH https://api.chat2cash.com/api/orders/ord_abc123/edit \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{

    "delivery_date": "2026-03-10T14:00:00Z",
    "special_instructions": "Call before delivery"
  }'
```

### Correct customer details

```bash
curl -X PATCH https://api.chat2cash.com/api/orders/ord_abc123/edit \
```

```bash
curl -X PATCH https://api.chat2cash.com/api/orders/ord_abc123/edit \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [

        "product_name": "Updated Item 1",
        "quantity": 3,
        "price": 100
      },
      {
        "product_name": "Updated Item 2",
        "quantity": 1,
        "price": 200
      }
    ],
    "total": 500
  }'
```

## Notes

- This endpoint uses **optimistic concurrency** - the update succeeds if the order exists and belongs to your organization
- Updating `items` performs a **delete-then-insert** operation, replacing all existing items
- The order must not be soft-deleted (`deletedAt` must be null)
- All numeric fields are validated (quantities must be positive, prices/totals must be non-negative)
- Field validation is strict - unknown fields in the request body will cause a validation error
