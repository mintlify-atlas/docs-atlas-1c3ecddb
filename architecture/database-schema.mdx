## Schema Overview

Chat2Cash uses **PostgreSQL 15** with **Drizzle ORM** for type-safe database access. The schema supports multi-tenant isolation via `organizationId` foreign keys on all data tables.

**ORM:** Drizzle ORM  
**Schema File:** `backend/src/schema.ts`  
**Migrations:** Drizzle Kit (run `npm run db:push` to sync schema)

## Entity Relationship Diagram

```
organizations (root tenant)
    |
    ├─► business_profiles (1:1)
    ├─► customers
    ├─► products
    ├─► orders
    │      └─► order_items
    ├─► users
    ├─► api_keys
    └─► roles
```

- `org_idx` on `organization_id` - Org-level queries

**Why JSONB for `raw_ai_response`:** Stores entire AI response for audit trails and debugging failed extractions.

**Why Soft Deletes:** `deleted_at` allows "undo" functionality and regulatory compliance (never fully delete order history).

1. User logs in via Neon Auth (JWT issued)
2. API verifies JWT and upserts user to `users` table (`backend/src/middlewares/authHandler.ts:71-85`)
3. Admin assigns `organization_id` via dashboard
4. Subsequent requests set `req.orgId = user.organizationId` (`backend/src/middlewares/authHandler.ts:90-92`)

**JIT User Provisioning:** Users are automatically created on first login using `INSERT ... ON CONFLICT DO UPDATE`.

### `api_keys`

**Purpose:** API key authentication for machine-to-machine access.

```typescript
export const apiKeysTable = pgTable("api_keys", {
  id: text("id").primaryKey(),
  organizationId: text("organization_id").references(() => organizationsTable.id).notNull(),
  keyHash: text("key_hash").notNull(),
  name: text("name").notNull(),
  maskedKey: text("masked_key").notNull(),
  isActive: boolean("is_active").default(true).notNull(),
  lastUsedAt: timestamp("last_used_at", { mode: 'string' }),
  createdAt: timestamp("created_at", { mode: 'string' }).defaultNow().notNull(),

  return {
    hashIdx: index("api_key_hash_idx").on(table.keyHash),
  };
});
```

**Code:** `backend/src/schema.ts:223-236`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `text` | PRIMARY KEY | API key ID |
| `organization_id` | `text` | FOREIGN KEY → `organizations.id` | Owning organization |
| `key_hash` | `text` | NOT NULL, INDEXED | SHA-256 hash of API key |
| `name` | `text` | NOT NULL | Human-readable key name |
| `masked_key` | `text` | NOT NULL | Masked key for display (e.g., `sk-...abc123`) |
| `is_active` | `boolean` | DEFAULT TRUE | Key revocation flag |
| `last_used_at` | `timestamp` | NULLABLE | Last usage timestamp |
| `created_at` | `timestamp` | DEFAULT NOW() | Creation timestamp |

**Indexes:**
- `api_key_hash_idx` on `key_hash` - Fast key lookup during authentication

**Security:**
- **Never stored plaintext:** Only SHA-256 hash is stored
- **Constant-time comparison:** Uses `crypto.timingSafeEqual()` to prevent timing attacks (`backend/src/middlewares/authHandler.ts:48-52`)
- **Active flag:** Revoked keys rejected at auth layer

**Authentication Flow:**

```typescript
1. Client sends: X-API-Key: sk-abc123...
2. API computes: SHA-256(sk-abc123...)
3. API queries: SELECT * FROM api_keys WHERE key_hash = <hash> AND is_active = true
4. API compares: crypto.timingSafeEqual(storedHash, incomingHash)
5. API sets: req.orgId = apiKey.organizationId
```

**Code:** `backend/src/middlewares/authHandler.ts:27-56`

## Data Types & Precision

### Numeric Columns

| Column | Type | Precision | Use Case |
|--------|------|-----------|----------|
| `tax_rate` | `numeric(5,2)` | 2 decimal places | Tax percentages (e.g., `18.00%`) |
| `total_amount` | `numeric(12,2)` | 2 decimal places | Currency (e.g., `1234567.89`) |
| `price_per_unit` | `numeric(12,2)` | 2 decimal places | Currency |
| `total_price` | `numeric(12,2)` | 2 decimal places | Currency |
| `quantity` | `numeric(12,3)` | 3 decimal places | Fractional quantities (e.g., `2.500 kg`) |

**Why `numeric` over `float`?** Avoids floating-point rounding errors in financial calculations.

### Timestamp Columns

**All timestamps use `{ mode: 'string' }` in Drizzle:**

```typescript
timestamp("created_at", { mode: 'string' })
```

- Returns ISO 8601 strings (e.g., `2025-03-04T12:34:56.789Z`)
- Avoids timezone conversion issues
- Compatible with JSON serialization

### JSONB Columns

| Column | Type | Description |
|--------|------|-------------|
| `raw_ai_response` | `jsonb` | Full Anthropic API response |
| `raw_messages` | `jsonb` | Array of chat messages |
| `invoice` | `jsonb` | Generated invoice JSON |
| `permissions` | `jsonb` | Array of permission strings |

**Why JSONB?**

- **GIN Indexing:** Fast full-text search (`@>`, `?`, `?|` operators)
- **Schema Flexibility:** AI responses change over time
- **No Joins:** Denormalized data for fast reads

## Multi-Tenant Isolation

### Row-Level Security Pattern

**Every query includes `organization_id` in WHERE clause:**

```typescript
// Example: Fetch orders for current org
const orders = await db
  .select()
  .from(ordersTable)
  .where(eq(ordersTable.organizationId, req.orgId)); // ← Tenant filter
```

**Why No PostgreSQL RLS Policies?**

- Application-level enforcement is simpler for multi-tenant SaaS
- Drizzle ORM makes it easy to enforce in code
- Future: Can add PostgreSQL RLS as defense-in-depth

### Foreign Key Cascade Rules

**No cascading deletes configured:**

- Deleting an organization requires explicit cleanup
- Prevents accidental data loss
- Allows soft deletes (`deleted_at` column)

## Sample Queries

### Fetch Orders with Items

```typescript
import { db } from './config/db';
import { ordersTable, orderItemsTable } from './schema';
import { eq } from 'drizzle-orm';

const ordersWithItems = await db

  .leftJoin(orderItemsTable, eq(ordersTable.id, orderItemsTable.orderId))
```

### Create Order with Items (Transaction)

```typescript
await db.transaction(async (tx) => {
  // Insert order
  const [order] = await tx.insert(ordersTable).values({
    id: 'ord_123',
    organizationId: 'org_abc',
    customerId: 'cust_xyz',
    // ...
  }).returning();

  // Insert items
  await tx.insert(orderItemsTable).values([
    { id: 'item_1', orderId: order.id, productName: 'Tomato', quantity: '10' },
    { id: 'item_2', orderId: order.id, productName: 'Onion', quantity: '5' },
  ]);
});
```

### Soft Delete Order

```typescript
await db
  .update(ordersTable)
  .set({ deletedAt: new Date().toISOString() })
```

## Performance Considerations

### Query Performance

- **Indexed lookups:** ~1-5ms (B-tree indexes)
- **Full-text search:** ~10-50ms (GIN indexes)
- **Joins:** ~5-20ms (foreign key indexes)

### Connection Pooling

Drizzle ORM uses PostgreSQL connection pooling:

```typescript
import { drizzle } from 'drizzle-orm/node-postgres';
import { Pool } from 'pg';

const pool = new Pool({
  connectionString: env.DATABASE_URL,
  max: 20, // Max 20 connections
});

export const db = drizzle(pool);
```

|--------|-------|-------|
| Max connections | 100 | PostgreSQL default |
| Table size | 10M+ rows | GIN indexes keep queries fast |

## Security Best Practices

### SQL Injection Prevention

**Drizzle ORM uses parameterized queries:**

```typescript
// Safe: Parameters are escaped
const orders = await db
  .select()
  .from(ordersTable)
  .where(eq(ordersTable.id, userInput));

// Dangerous: Raw SQL (avoid unless necessary)
const results = await db.execute(sql`SELECT * FROM orders WHERE id = ${userInput}`);
```

### Secrets Management

**Never commit to git:**

- `DATABASE_URL`
- API keys
- JWT secrets

Use environment variables (`.env` file + Docker Compose secrets).

### Audit Logging

**Track changes with timestamps:**

- `created_at` on all tables
- `updated_at` on `business_profiles`
- `deleted_at` for soft deletes

**Future:** Add `audit_log` table for compliance.
